## 기존
insert_ok2.jsp , find_ok2.jsp 등과 같이 개발하는 경우는 없다. 오류가 났을 때 어디서 오류가 났는지 명확하지 않다.(디자이너의 잘못인가, 프로그래머의 잘못인가?)

## 자바빈 액션으로 해결
1. 자바빈을 사용하게 되면 디자인 부분과 로직부분을 나누어서 개발을 할 수 있음 => 디자이너는 디자인만, 프로그래머는 프로그램만 나누어서 개발을 하기 위해 만든 방식
2. 설계규약
  1) 멤버변수마다 별도의 (table 관련 필드값과 연관된) get/set 메소드가 존재해야 한다.(메소드명은 멤버변수와 일치해야 함) ex) DTO
  2) get 메소드의 매개변수(파라미터)는 존재하면 안됨.
  3) set 메소드는 반드시 하나 이상의 매개변수 존재해야 함.
  4) beans 컴포넌트의 속성은 반드시 읽기 또는 쓰기가 가능해야 함. ex) public 으로 만들어야 함.
  5) 생성자는 매개변수(파라미터)가 존재하지 않아야 한다. -> 불러낼 수 있어야 하므로.
3. <jsp:useBean/> 태그
  > 자바빈 객체를 생성해 주기 위한 태그
  ~~~
  <jsp:useBean id="빈이름" class="자바빈 클래스 명" scope="사용범위" />
  ~~~
  이때, scope 의 값은 **1)page 2)request 3)session 4)application** 으로 총 4가지가 있다.
4. <jsp:setProperty/> 태그
  > 자바빈 클래스의 속성값을 설정하기 위한 태그
  ~~~
  <jsp:setProperty name="빈이름" property="속성명" value="설정할 속성값"/>
  ~~~
5. <jsp:getProperty/> 태그
  > 자바빈 클래스의 속성값을 가져오기 위한 태그
  ~~~
  <jsp:getProperty name="빈이름" property="속성명" />
  ~~~
6. DTO와 DAO의 개념
  1) DTO
    * Data(Table) Transfer Object(객체)
    * 테이블의 구조와 똑같은 객체를 만들어 줌
  2) DAO
    * Data Access Object
    * 테이블의 자료를 Access 하는 객체를 만들어 줌
